1.6.9 – UX Specification (Improvement Layer)
________________________________________
1. Objective
Define the UX for the Improvement Layer (1.6) to enable:
•	Structured RCA execution
•	Clear CAPA management
•	Seamless PDCA lifecycle progression
•	Strong traceability to 1.5 (Execution)
•	Fast adoption by shop-floor + supervisors + quality teams
________________________________________
2. UX Design Principles (Aligned with 1.2–1.5)
________________________________________
2.1 Create vs Use Separation
•	Improvement cases are not created standalone
•	Created ONLY from:
o	Abnormality (1.5)
o	Escalation
o	Repeat detection
________________________________________
2.2 No Duplication
•	SOP → from 1.4
•	Measure → from 1.3
•	Owner → from 1.2
👉 UI only references, never duplicates
________________________________________
2.3 Progressive Disclosure
•	Show only relevant fields per stage:
o	RCA → hide CAPA
o	CAPA → hide validation
o	Validation → summary view
________________________________________
2.4 Traceability First
User can navigate:
Case → Abnormality → SOP Step → Measure → Owner
(≤ 3 clicks)
________________________________________
2.5 Action-Oriented UX
•	No passive screens
•	Every screen drives:
o	RCA
o	CAPA
o	Validation
