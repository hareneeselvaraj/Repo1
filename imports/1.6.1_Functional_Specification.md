1.6.1 – Functional Specification (Improvement Layer)
________________________________________
1. Objective
The objective of the Improvement Layer is to:
•	Identify and eliminate root causes of recurring or critical issues
•	Prevent recurrence through structured corrective and preventive actions
•	Continuously improve processes and standards
________________________________________
Core Principle
Do not solve every problem deeply.
Solve recurring and critical problems permanently.
________________________________________
2. Scope
________________________________________
2.1 In Scope
•	Root Cause Analysis (RCA)
•	Corrective and Preventive Actions (CAPA)
•	PDCA lifecycle management
•	Structured problem-solving (5 Why, Fishbone)
•	Improvement case tracking
•	Standardization feedback loop
________________________________________
2.2 Out of Scope
•	Abnormality detection → handled in 1.5
•	Immediate corrective actions → handled in 1.5
•	KPI monitoring → handled in 1.5
________________________________________
3. Entry Criteria (LOCKED)
________________________________________
Improvement Case can be created ONLY when:
•	Repeated abnormality detected
•	High severity issue
•	Escalated from higher tier
•	User explicitly selects “Escalate to RCA”
________________________________________
Source
Abnormality (1.5) → Disposition → Escalate to RCA → Improvement Case
________________________________________
4. Functional Overview
________________________________________
The Improvement Layer is a guided problem-solving system that transforms:
Problem → Root Cause → Action → Standardization → Prevention
________________________________________
________________________________________
5. Core Functional Capabilities
________________________________________
🔷 5.1 Improvement Case Management
________________________________________
Description
Central entity representing a problem requiring deep investigation.
________________________________________
Capabilities
•	Create from abnormality (pre-filled)
•	Assign owner (Position → Person)
•	Track lifecycle
•	Maintain full traceability
________________________________________
________________________________________
🔷 5.2 PDCA Lifecycle Management
________________________________________
Stages
Stage	Description
Plan	Define problem, analyze cause
Do	Implement corrective action
Check	Validate effectiveness
Act	Standardize and close
________________________________________
________________________________________
Capabilities
•	Track stage progression
•	Capture inputs per stage
•	Enforce structured flow
________________________________________
________________________________________
🔷 5.3 Root Cause Analysis (RCA)
________________________________________
Supported Methods
•	5 Why
•	Fishbone (Ishikawa)
•	Cause Tree (optional)
________________________________________
________________________________________
Capabilities
•	Capture root cause hierarchy
•	Link cause to problem
•	Document findings
________________________________________
________________________________________
🔷 5.4 CAPA Management
________________________________________
Types
Type	Purpose
Corrective	Fix current issue
Preventive	Avoid recurrence
________________________________________
________________________________________
Capabilities
•	Create CAPA from RCA
•	Assign ownership
•	Track execution
•	Link to root cause
________________________________________
________________________________________
🔷 5.5 Effectiveness Validation
________________________________________
Description
Verify whether implemented actions resolved the issue.
________________________________________
Capabilities
•	Compare KPI before/after
•	Validate no recurrence
•	Mark as effective / ineffective
________________________________________
________________________________________
🔷 5.6 Standardization Feedback Loop
________________________________________
MOST CRITICAL CAPABILITY
________________________________________
Improvement → Update SOP / Measure → Reflect in Execution
________________________________________
________________________________________
Capabilities
•	Suggest SOP update (1.4)
•	Suggest measure update (1.3)
•	Track change impact
________________________________________
________________________________________
🔷 5.7 Yokoten (Knowledge Sharing)
________________________________________
Description
Replicate improvements across similar contexts.
________________________________________
Capabilities
•	Identify similar contexts (lines/sites)
•	Recommend replication
•	Track adoption
________________________________________
________________________________________
🔷 5.8 Improvement Traceability
________________________________________
End-to-End Link
KPI → Abnormality → Action → RCA → CAPA → SOP Update → KPI
________________________________________
________________________________________
6. User Roles and Responsibilities
________________________________________
Role	Responsibility
Supervisor	Initiate improvement
Manager	Own RCA and CAPA
Quality Team	Validate root cause
Leadership	Review major improvements
________________________________________
________________________________________
7. End-to-End Workflow
________________________________________
1. Abnormality detected (1.5)
2. Disposition → Escalate to RCA
3. Improvement case created
4. Root cause analysis
5. CAPA defined
6. CAPA executed
7. Effectiveness validated
8. SOP / Measure updated
9. Case closed
________________________________________
________________________________________
8. Constraints
________________________________________
•	No standalone improvement creation
•	No skipping RCA stage
•	No CAPA without root cause
•	Must link to execution context
________________________________________
________________________________________
9. Success Criteria
________________________________________
Improvement Layer is successful when:
•	Only meaningful cases are created
•	Root causes are clearly identified
•	CAPA prevents recurrence
•	SOP updates reflect improvements
•	System learns over time
________________________________________
________________________________________
10. Key Differentiator
________________________________________
Your system becomes:
Execution System (1.5)
+ Learning System (1.6)
